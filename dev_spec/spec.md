### 1. 변환 및 포맷팅 도구 (The Basics)
가장 기본적이지만, 의외로 매번 구글링하게 되는 기능들입니다. Shadcn의 `Textarea`와 `Button`만으로도 훌륭하게 구현할 수 있습니다.

* **JSON Formatter & Validator (+ Diff)**
    * **기능:** 더러운 JSON 문자열을 예쁘게 정렬하고, 문법 오류를 잡아줍니다.
    * **아이디어:** 두 개의 JSON을 입력받아 **차이점(Diff)**을 하이라이팅해 주는 기능은 API 버전 변경 시 응답값을 비교할 때 매우 유용합니다.
* **Java Class ↔ JSON 변환기**
    * **기능:** 프론트엔드 요청에 맞춰 DTO를 만들거나, 반대로 DTO를 보고 JSON 예시를 만들 때 사용합니다.
    * **포인트:** Java의 `CamelCase`와 DB/JSON의 `Snake_case`를 상호 변환해 주는 옵션을 넣으면 인기 만점입니다.
* **YAML ↔ Properties 변환기 (Spring Boot 필수)**
    * **이유:** Spring Boot 설정 파일(`application.yml` vs `application.properties`) 형식을 서로 바꿔야 할 때가 빈번합니다. 온라인 툴이 있지만, 보안상 사내 설정(비번 등)을 외부 사이트에 붙여넣기 껄끄러울 때 유용합니다.

### 2. 시간 및 데이터 처리 (Time & Data)
* **Epoch/Unix Timestamp 변환기**
    * **기능:** DB에 저장된 `1712345678` 같은 숫자를 `2025-04-05 12:00:00` 같은 사람이 읽을 수 있는 시간으로 변환합니다.
    * **심화:** 밀리초(ms)와 초(s) 단위를 자동 감지하거나, **KST(한국 표준시)와 UTC를 동시에 보여주는 기능**을 넣으세요.
* **Cron Expression 생성기/테스트**
    * **이유:** Spring Scheduler나 Python Celery Beat 설정할 때 크론 표현식(`0 0/5 * * * ?`)은 항상 헷갈립니다.
    * **기능:** GUI로 분/시/일을 선택하면 크론 식을 생성해주고, "다음 실행 예정 시간 10개"를 미리 보여줍니다.

### 3. 보안 및 인증 (Security & Auth)
* **JWT 디코더 (오프라인 버전)**
    * **이유:** `jwt.io`가 유명하지만, 사내 토큰(민감 정보 포함)을 외부망에 올리는 건 보안 위반일 수 있습니다.
    * **기능:** 순수 브라우저 로직(Client-side JS)으로만 JWT를 파싱해서 Header와 Payload 내용을 보여줍니다. (Shadcn의 `Card` 컴포넌트로 예쁘게 보여주기 좋습니다.)
* **URL Encode / Decode**
    * **기능:** 한글이나 특수문자가 포함된 쿼리 파라미터가 깨질 때 확인용입니다.

### 4. 데이터베이스 및 쿼리 헬퍼 (DB Helper)
* **SQL 파라미터 바인딩 (MyBatis/JPA 로그 변환)**
    * **고충:** Spring Boot 로그에서 `PreparedStatement` 쿼리는 `?` 로 표시되고 파라미터는 뒤에 따로 나옵니다. 디버깅할 때 이걸 합쳐서 완전한 SQL로 보고 싶어 합니다.
    * **기능:** 로그 텍스트(SQL문과 파라미터)를 붙여넣으면, `?` 자리에 값을 채워 넣어 바로 실행 가능한 SQL 문장으로 만들어줍니다.
* **Redis Key 패턴 스캐너 (모의)**
    * **기능:** 실제 운영 DB에 붙기는 위험하더라도, 로컬이나 개발계 Redis의 Key 네이밍 컨벤션을 검증하거나, 자주 쓰는 Key 패턴을 문서화해두고 클릭 시 값을 조회하는 명령어(`get user:1:info`)를 복사해주는 도구입니다.

### 5. 사내 특화 도구 (Team Specific - ★강추★)
이 부분이 실제로 동료들이 가장 고마워할 영역입니다.

* **에러 코드 조회기**
    * **기능:** 프로젝트 내에서 정의한 커스텀 에러 코드(예: `ERR_User_001`)와 그 의미, 해결 방법을 검색할 수 있게 합니다. 코드를 뒤지는 것보다 훨씬 빠릅니다.
* **정규식(RegEx) 테스트 & 라이브러리**
    * **기능:** 이메일, 전화번호, 비밀번호 규칙 등 사내에서 공통으로 사용하는 정규식을 미리 저장해두고 테스트할 수 있는 공간입니다.

1.  **라이브러리 활용:**
    * 시간 변환: `date-fns` 또는 `dayjs`

2.  **클라이언트 사이드 처리:**
    * 보안을 위해 데이터 변환 로직은 **서버로 보내지 말고 브라우저 내(Client Component)에서 처리**되도록 만드세요. (`"use client"` 지시어 사용)
3.  **UI/UX:**
    * Shadcn의 `Tabs` 컴포넌트를 사용해 하나의 페이지에서 [JSON] | [YAML] | [XML] 탭을 오가며 변환하게 하면 깔끔합니다.
    * `Toast` (Sonner) 기능을 넣어 변환된 텍스트를 클릭 한 번에 클립보드에 복사되게 하세요. (개발자 경험 상승!)
